# Building & installing the hevyier APK

Fully local, offline. No Expo account, no cloud build, no OTA.
(See plan decision #1, revised 2026-06-13.)

## Prerequisites (one-time, on the build machine)

- **JDK 17** (`java -version`).
- **Android SDK** + platform-tools (`adb`). Easiest via Android Studio, or set
  `ANDROID_HOME` to an SDK install. `adb` must be on `PATH`.
- USB debugging enabled on the phone (Settings → Developer options).

## Build

```bash
npm run apk
```

That runs:

```bash
expo prebuild --platform android   # regenerates the gitignored android/ dir from app.json
cd android && ./gradlew assembleRelease
```

Output APK:

```
android/app/build/outputs/apk/release/app-release.apk
```

`android/` is gitignored (continuous native generation) — it is regenerated
from `app.json` every build, so never hand-edit files inside it.

## Install

```bash
adb install -r android/app/build/outputs/apk/release/app-release.apk
```

`-r` reinstalls over the existing app and **keeps the SQLite database** (your
logged workouts survive updates), as long as the signing key is unchanged.

No cable? Copy the APK to the phone and tap it (allow "install from unknown
sources" once).

## Signing

By default the release build is signed with the machine's **debug keystore**
(`~/.android/debug.keystore`, auto-created). For solo sideloading this is fine:

- Rebuilds on the **same machine** share that key → updates install in place.
- Rebuilds on a **different machine** use a different debug key → Android
  refuses the upgrade; uninstall first (this wipes the DB).

### Optional: portable stable key

Only needed if you build on more than one machine and want in-place upgrades.

```bash
# 1. generate once, store OUTSIDE the repo (keystores are gitignored anyway)
keytool -genkey -v -keystore ~/keys/hevyier.keystore \
  -alias hevyier -keyalg RSA -keysize 2048 -validity 10000

# 2. put credentials in ~/.gradle/gradle.properties (global, never in the repo)
#    HEVYIER_STORE_FILE=/home/<you>/keys/hevyier.keystore
#    HEVYIER_KEY_ALIAS=hevyier
#    HEVYIER_STORE_PASSWORD=...
#    HEVYIER_KEY_PASSWORD=...
```

Then add a release `signingConfig` reading those properties. Because `android/`
is regenerated by prebuild, wire it through an Expo config plugin
(`withAppBuildGradle`) rather than editing the generated `build.gradle` — see
the Expo "Continuous Native Generation" docs. Skip this entirely unless you
actually hit the multi-machine case.

## Versioning

Bump `version` in **both** `package.json` and `app.json` together for any
native/SDK change, then rebuild + reinstall. JS-only changes don't strictly
need a bump (no OTA — you reinstall regardless), but bumping keeps the installed
"about" version honest.

## Smoke checklist (after install)

Fresh install seeds DB → build a plan → assign a weekday → run a full session
with the rest timer (lock the phone, confirm the vibrating notification) → edit
a set from History → open Analytics (charts render) → kill the app mid-session →
reopen and confirm Resume works.
